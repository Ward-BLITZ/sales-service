Mailbox koppeling
Verbind een e-mailaccount via IMAP/SMTP zodat je mails kan ontvangen en versturen vanuit de tool. De mails worden via een Supabase Edge Function gesynchroniseerd.
IMAP (ontvangen)
SMTP (versturen)
Outlook / Microsoft 365 (ward@blitzpower.com):
IMAP: outlook.office365.com poort 993 (SSL)
SMTP: smtp.office365.com poort 587 (STARTTLS)
Gebruikersnaam = je volledige e-mailadres. Wachtwoord = je normale Microsoft wachtwoord.
Als je organisatie MFA gebruikt: ga naar security.microsoft.com → App passwords om een app-wachtwoord aan te maken.
Configuratie wordt opgeslagen in Supabase (service_settings).

Database tabellen aanmaken
Voer dit eenmalig uit in de Supabase SQL Editor:
CREATE TABLE IF NOT EXISTS service_tickets (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  data jsonb NOT NULL,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);
CREATE TABLE IF NOT EXISTS service_interventies (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  data jsonb NOT NULL,
  created_at timestamptz DEFAULT now()
);
CREATE TABLE IF NOT EXISTS service_mails (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  data jsonb NOT NULL,
  created_at timestamptz DEFAULT now()
);
CREATE TABLE IF NOT EXISTS service_settings (
  key text PRIMARY KEY,
  data jsonb NOT NULL
);
ALTER TABLE service_tickets ENABLE ROW LEVEL SECURITY;
ALTER TABLE service_interventies ENABLE ROW LEVEL SECURITY;
ALTER TABLE service_mails ENABLE ROW LEVEL SECURITY;
ALTER TABLE service_settings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "allow all" ON service_tickets FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all" ON service_interventies FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all" ON service_mails FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "allow all" ON service_settings FOR ALL USING (true) WITH CHECK (true);
